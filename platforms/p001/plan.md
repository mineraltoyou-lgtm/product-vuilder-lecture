# [P001] WHO 글로벌 원격 의료 플랫폼: Living Design Document

## 제 1조: CEO 리뷰 (Founder/CEO Persona) - /plan-ceo-review

> "우리가 만드는 것이 '또 하나의 원격 진료 앱'이 아니라는 것을 10-star product 관점에서 증명해야 합니다."

### 1. 우리는 어떤 문제를 해결하는가? (Problem)
- **Problem:** 개발도상국의 수십억 인구는 전문 의료 서비스에 대한 접근성이 극도로 제한되어 있으며, 이는 예방 가능한 질병으로 인한 높은 사망률로 이어진다. 의사 한 명을 양성하는 데는 10년 이상이 걸리지만, AI는 이 시간과 비용의 장벽을 뛰어넘을 잠재력을 가지고 있다.
- **Root Cause:** 물리적 인프라(병원), 전문 인력(의사), 경제적 비용의 삼중 장벽.
- **So What?:** 이는 단순한 불편함을 넘어, 수많은 생명을 앗아가는 구조적인 불평등의 문제이다.

### 2. 우리의 솔루션은 무엇인가? (Solution)
- **Solution:** 스마트폰 카메라와 간단한 의료 센서(옵션)를 활용하여, AI가 1차적인 질병 스크리닝 및 진단 보조를 수행하는 **'주머니 속의 AI 의사'**를 제공한다. 이는 현지 보건 요원(CHW)의 역량을 강화하고, 전문의의 개입이 시급한 환자를 정확하게 분류(Triage)하는 것을 목표로 한다.
- **Core Function:**
    1.  **AI 영상 진단:** 피부 질환, 안구 질환 등 스마트폰 카메라로 촬영된 이미지를 AI가 분석하여 질병 가능성을 제시한다.
    2.  **AI 문진 챗봇:** 환자의 증상을 바탕으로 대화형 문진을 수행하여 위험도를 평가한다.
    3.  **데이터 대시보드:** 지역별 질병 발병 현황을 시각화하여, 공공 보건 정책 수립을 위한 실시간 데이터를 제공한다.

### 3. 이 제품이 왜 '10-star' 경험인가? (10-star Experience)
- **1-star:** 그냥 원격 진료 앱.
- **3-star:** AI가 사진을 보고 간단한 피부병을 알려줌.
- **5-star:** AI가 문진과 사진 분석을 결합하여 여러 질병의 가능성을 %로 알려주고, 현지 보건 요원에게 프로토콜을 제시함.
- **7-star:** 환자의 과거 의료 기록, 지역 발병 데이터까지 통합 분석하여 맞춤형 건강 리포트를 생성하고, 필요한 약품을 드론으로 배송해줌.
- **10-star:** **내 유전 정보와 생활 습관을 분석한 AI 주치의가, 질병이 발생하기 전에 이상 징후를 감지하고, 내 몸의 '디지털 트윈'을 통해 최적의 치료법을 시뮬레이션하여 나에게 제안한다.** 우리 MVP는 5-star를 목표로 하되, 10-star의 비전을 향한 확장성을 설계에 반드시 포함해야 한다.

### 4. 성공의 기준은 무엇인가? (Success Metrics)
- **과제 수주 관점 (단기):**
    - **Metric 1:** MVP 시연 시, 5대 주요 피부 질환에 대한 AI 진단 정확도 90% 이상 달성.
    - **Metric 2:** 심사위원들이 '이것은 인류를 위한 기술'이라고 감탄하게 만드는 명확하고 강력한 시나리오.
- **비즈니스 관점 (장기):**
    - **Metric 1:** 현지 보건 요원(CHW) 1인당 하루 처리 가능 환자 수 2배 증가.
    - **Metric 2:** AI 스크리닝을 통한 중증 질환 조기 발견율 20% 향상.

### 5. 우리는 무엇을 '하지 않을 것'인가? (Non-Goals)
- **하지 않을 것 1:** AI가 최종 진단을 내리는 것. 모든 진단은 '보조'의 역할이며, 최종 책임은 의료 전문가에게 있음을 명확히 한다.
- **하지 않을 것 2:** 자체 의료 하드웨어를 개발하는 것. 기존의 스마트폰과 상용 센서를 최대한 활용하여 접근성을 높인다.
- **하지 않을 것 3:** 개인 식별이 가능한 의료 정보를 중앙 서버에 저장하는 것. 데이터 익명화와 분산 저장을 원칙으로 한다.

## 제 2조: 엔지니어링 리뷰 (Eng Manager Persona) - /plan-eng-review

> "비전은 아키텍처로 증명해야 합니다. 모든 요구사항을 만족시키면서, 가장 단순하고, 확장 가능하며, 기술 부채가 없는 시스템을 설계합니다."

### 1. 시스템 아키텍처 다이어그램 (System Architecture)

```
[End User: CHW/Patient] <--> [Client: PWA] <--> [API Gateway] <--> [Backend: Serverless Functions] <--> [AI/ML Service]
       |                                                 |                     |                         |
       |                                                 |                     |                         +--> [Database: NoSQL]
       |                                                 |                     |
       +-------------------------------------------------+---------------------+--> [Data Lake/Warehouse] --> [Dashboard: Metabase/Looker]
```

-   **Client (PWA - Progressive Web App):** 오프라인 사용성과 낮은 사양의 디바이스를 지원하기 위해 네이티브 앱이 아닌 PWA를 채택합니다. `platforms/p001/index.html`이 이 PWA의 엔트리 포인트가 됩니다.
-   **Backend (Serverless Functions):** 예측 불가능한 트래픽에 대응하고, 초기 인프라 비용을 최소화하기 위해 서버리스 아키텍처를 사용합니다. (e.g., Firebase Functions, AWS Lambda)
-   **AI/ML Service (TensorFlow Lite / Vertex AI):** 모델 경량화를 통해 일부 추론은 클라이언트(On-Device AI)에서 직접 수행하여 속도를 높이고, 복잡한 분석은 서버에서 처리하는 하이브리드 접근 방식을 취합니다.
-   **Database (NoSQL):** 비정형 의료 데이터(이미지, 텍스트)를 유연하게 저장하기 위해 NoSQL DB(e.g., Firestore, DynamoDB)를 사용합니다.

### 2. 데이터 흐름 (Data Flow)

1.  **데이터 수집 (Ingestion):** 사용자가 PWA를 통해 이미지와 증상 텍스트를 업로드합니다. 모든 데이터는 전송 계층 보안(TLS)으로 암호화됩니다.
2.  **데이터 익명화 (Anonymization):** API 게이트웨이 또는 백엔드 함수에서 개인식별정보(PII)를 제거하고, 고유 식별자(UUID)를 생성하여 데이터를 분리 저장합니다. 원본 이미지는 즉시 폐기하거나, 별도의 보안 스토리지에 암호화하여 저장합니다.
3.  **AI 추론 (Inference):** 익명화된 데이터가 AI 서비스로 전달되어 분석됩니다.
4.  **결과 반환 (Result):** 분석 결과(질병 가능성, 위험도)가 사용자에게 반환됩니다. 이 결과는 '진단'이 아닌 '참고 정보'임을 명확히 고지합니다.
5.  **데이터 집계 (Aggregation):** 익명화된 데이터는 데이터 레이크로 안전하게 이동하여, 공공 보건 대시보드를 위한 통계 자료로만 활용됩니다.

### 3. 기술 스택 (Tech Stack) - MVP 기준

-   **Frontend:** HTML5, CSS3, Vanilla JavaScript (프레임워크 없이 경량화에 집중) -> **PWA**
-   **Backend:** Firebase Functions (Node.js/Python)
-   **Database:** Cloud Firestore
-   **AI Model:** TensorFlow Lite (On-Device), Google Vertex AI (Server-side)
-   **Storage:** Google Cloud Storage
-   **Hosting:** Firebase Hosting

### 4. 에지 케이스 및 리스크 관리 (Edge Cases & Risk Mitigation)

-   **Edge Case 1: 오프라인 환경**
    -   **Mitigation:** PWA의 서비스 워커(Service Worker)를 활용하여 AI 문진 기능과 과거 기록 조회를 오프라인 상태에서도 가능하게 합니다. 이미지 업로드 등은 온라인 전환 시 자동으로 동기화됩니다.
-   **Edge Case 2: 저품질 이미지**
    -   **Mitigation:** 이미지 업로드 시, 클라이언트 단에서 해상도, 흔들림, 조명 상태를 사전 체크하여 사용자에게 가이드를 제공합니다. AI 모델은 저품질 데이터에 대한 강건성(Robustness) 훈련을 병행합니다.
-   **Edge Case 3: 의료법 및 데이터 규제 (HIPAA, GDPR 등)**
    -   **Mitigation:** MVP 단계부터 가장 엄격한 수준의 데이터 익명화 조치를 적용합니다. (CEO Non-Goal 3). 법률 자문을 통해 각 진출 국가의 규제 준수 여부를 철저히 검증합니다.
-   **Risk: AI 모델의 편향성 (Bias)**
    -   **Mitigation:** 특정 인종, 성별에 대한 데이터 쏠림 현상을 방지하기 위해, 다양한 인종 데이터를 포함하는 공개 데이터셋(e.g., ISIC, DermNet)과 파트너 기관의 데이터를 활용하여 모델을 훈련하고, 지속적인 편향성 감사를 수행합니다.

## 제 3조: 디자인 리뷰 (Senior Designer Persona) - /plan-design-review

> "가장 뛰어난 기술은 가장 인간적인 얼굴을 하고 있습니다. 우리는 'AI Slop'을 제거하고, 모든 픽셀에 신뢰와 공감을 담아내야 합니다."

### 1. 디자인 핵심 원칙 (Core Design Principles)

1.  **신뢰 최우선 (Trust First):** 모든 시각적 요소는 전문성과 안정감을 전달해야 합니다. WHO의 공식 가이드라인을 연상시키는 절제된 컬러 팔레트(파란색, 흰색, 회색 계열)를 사용하고, 명확하고 간결한 언어를 사용합니다. AI가 제시하는 모든 정보에는 **"이것은 의료적 진단이 아닌, 참고 정보입니다."** 라는 경고 문구를 명확하고 상시적으로 노출하여 사용자의 오해를 방지합니다.
2.  **극도의 단순성 (Radical Simplicity):** 디지털 기기에 익숙하지 않은 사용자를 위해, 하나의 화면에는 하나의 핵심 기능만 배치합니다. 아이콘은 텍스트 레이블과 함께 사용하여 오해의 소지를 없애고, 모든 버튼은 크고 명확하게 디자인하여 터치 실수를 줄입니다.
3.  **안내하는 경험 (Guided Experience):** 사용자가 다음에 무엇을 해야 할지 고민하지 않도록, 모든 프로세스는 단계별로 명확하게 안내되어야 합니다. 특히 사진 촬영 시에는, "조금 더 밝은 곳에서 촬영해주세요" 또는 "환부가 중앙에 오도록 맞춰주세요" 와 같이 구체적인 피드백을 실시간으로 제공하여 성공률을 높입니다.
4.  **문화적 보편성 (Cultural Universality):** 특정 문화권에서만 이해되는 아이콘, 제스처, 색상 사용을 지양합니다. 모든 시각 언어는 보편적으로 이해 가능해야 하며, 필요시 현지 문화에 대한 철저한 검증을 거칩니다.

### 2. MVP 핵심 화면 설계 (Key Screen Wireframes)

-   **Screen 1: 홈 (Home)**
    -   두 개의 거대한 버튼만 존재: `[새로운 환자 기록 시작하기]` / `[기존 환자 기록 보기]`
    -   상단에는 WHO 로고와 함께 기관의 공식 명칭을 표기하여 신뢰감을 줍니다.
-   **Screen 2: 사진 촬영 가이드 (Image Capture Guide)**
    -   화면 중앙에 사람의 신체 부위(피부, 눈)를 형상화한 오버레이 가이드를 표시합니다.
    -   실시간 이미지 품질 분석 결과(밝기, 흔들림)를 화면 상단에 간단한 아이콘(☀️, 揺)으로 표시합니다.
-   **Screen 3: AI 분석 결과 (AI Result)**
    -   **가장 중요한 화면.** 복잡한 차트나 그래프를 지양하고, **신호등 시스템(Red/Yellow/Green)**을 사용하여 위험도를 직관적으로 표현합니다.
        -   **Red (위험):** "전문의의 진료가 시급합니다. 가장 가까운 협력 병원으로 즉시 안내합니다."
        -   **Yellow (주의):** "피부 질환의 가능성이 있습니다. 현지 보건 요원을 위한 다음 단계 프로토콜을 확인하세요."
        -   **Green (안전):** "심각한 질병의 징후는 보이지 않습니다. 하지만 증상이 지속되면 다시 확인해주세요."
    -   AI가 예측한 질병 후보군은 작은 글씨와 %로 표시하되, 항상 "예측 정보"임을 강조합니다.

### 3. 시각적 정체성 (Visual Identity) - MVP

-   **Color Palette:**
    -   Primary: #0056A6 (WHO 공식 블루 - 신뢰, 안정)
    -   Secondary: #FFFFFF (배경 - 명료성, 깨끗함)
    -   Accent (Signal): #D8412C (Red), #F7B829 (Yellow), #2E8540 (Green)
-   **Typography:** Noto Sans (다국어 지원 및 뛰어난 가독성)
-   **Iconography:** Material Design Icons (보편성 및 일관성)
